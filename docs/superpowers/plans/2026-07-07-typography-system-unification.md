# Typography System Unification Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Unify app-owned CSS and Naive UI typography behind one system font stack and shared text scale.

**Architecture:** Add typography tokens to the root CSS scope, then replace scattered font declarations with those tokens without changing layout dimensions. Mirror the same font stack and scale into Naive UI `themeOverrides.common`, and align the static fallback stylesheet with the same family.

**Tech Stack:** Vue 3, Naive UI, Vite, CSS custom properties, pnpm.

---

## File structure

- Modify: `frontend/src/styles.css`
  - Responsibility: app-level visual system and layout styles. This file will own CSS typography tokens and consume them for app-owned elements.
- Modify: `frontend/src/App.vue`
  - Responsibility: top-level Naive UI provider configuration. This file will mirror the app typography scale into `themeOverrides.common` for library components.
- Modify: `app/static/styles.css`
  - Responsibility: legacy/static fallback page styling. This file will only receive the shared system font stack.
- Do not modify: `app/static/dist/**`
  - Responsibility: generated build output. It will be regenerated by Vite when needed, not hand-edited.
